Our implementation from the session used 107 out of the 20800 available LUTs, and our implementation from this report used 125 out of the 20800 available LUTs.
Obviously, both used the same number of IO ports, that being 101 out of the available 106.
We conclude that implementing such common circuitry when the synthesis tool you use already chooses an optimized default implementation for you is a bad idea in most cases. It increases code complexity while probably not increasing performance, except you know what you are doing and want a specific implementation for some reason. For example, the explicit gate-level adder increased LUT usage, so it uses more FPGA area. Therefore, for this design, manually building the adder was less area-efficient than letting Vivado infer and optimize the adder.
